Remarks

If an AtomEntry is copied from one feed into another feed, then the source feed's metadata (all child elements of feed other than the entry elements) may be preserved within the copied entry by specifying an AtomSource, if it is not already present in the entry, and including some or all of the source feed's meta-data elements as the source's children. Such metadata should be preserved if the source feed contains any of the child elements author, contributor, rights, or category and those child elements are not present in the source entry.

The AtomSource is designed to allow the aggregation of entries from different feeds while retaining information about an entry's source feed. For this reason, Atom Processors that are performing such aggregation should include at least the required feed-level meta-data elements (id, title, and updated) in the AtomSource.
